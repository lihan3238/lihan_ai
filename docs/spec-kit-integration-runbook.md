# Spec Kit Integration Runbook

This project uses Superpowers for agent discipline, repo-native AI development gates, and GitHub Spec Kit as an upstream spec lifecycle tool layer. Spec Kit `v0.8.7` is initialized in Codex skills mode.

## Current Decision

- Superpowers remains the execution discipline layer: brainstorming, TDD, debugging, verification, and branch finishing.
- Repo-native `docs/templates/ai-dev/`, `ops/ai-dev-check.sh`, and `ops/production-gate.sh` remain the authoritative local project gates.
- GitHub Spec Kit is installed for Codex via `.agents/skills/speckit-*` and shared workflow assets in `.specify/`.

## Installed Assets

The repository now tracks:

- `.specify/`: Spec Kit scripts, templates, workflow registry, and constitution memory.
- `.agents/skills/speckit-*`: Codex skill entrypoints generated by Spec Kit.
- `AGENTS.md`: Spec Kit context marker file.

`.gitignore` ignores `.agents/**` by default and re-allows `.agents/skills/**`, so future private agent state is not accidentally committed.

## Prerequisites

Windows has `specify` available at `C:\Users\lihan\.local\bin\specify.exe`. Use UTF-8 output when running Spec Kit from PowerShell:

```powershell
$env:PYTHONIOENCODING="utf-8"
specify version
specify check
```

If GitHub or Docker Hub access is unstable from WSL, use the Windows host proxy:

```bash
export HTTP_PROXY=http://10.88.0.6:10808
export HTTPS_PROXY=http://10.88.0.6:10808
export http_proxy=http://10.88.0.6:10808
export https_proxy=http://10.88.0.6:10808
```

## Install Or Refresh The Pinned CLI

Use the GitHub source, not a same-named package from PyPI:

```bash
uv tool install specify-cli --from git+https://github.com/github/spec-kit.git@v0.8.7
specify version
specify check
```

If an older install exists:

```bash
uv tool install specify-cli --force --from git+https://github.com/github/spec-kit.git@v0.8.7
```

## Sandbox Initialization

For future upgrades, initialize a throwaway project before touching this repository:

```bash
mkdir -p tmp/spec-kit-smoke
cd tmp/spec-kit-smoke
specify init my-project --integration codex --integration-options="--skills" --script sh --ignore-agent-tools
```

Inspect generated files:

```bash
find my-project -maxdepth 3 -type f | sort
```

Expected areas to review:

- `.specify/`
- generated scripts
- generated templates
- generated Codex skills or command files
- any agent instruction files

## Repository Initialization

This repository was initialized with:

```powershell
$env:PYTHONIOENCODING="utf-8"
specify init --here --offline --integration codex --integration-options="--skills" --script sh --ignore-agent-tools --no-git --force
```

Use `--force` only after reviewing sandbox output or when reapplying the same pinned version.

## Codex Usage

Start Codex in the repository root and invoke the generated skills:

```text
$speckit-constitution
$speckit-specify
$speckit-plan
$speckit-tasks
$speckit-implement
```

Optional quality skills are also installed:

```text
$speckit-clarify
$speckit-analyze
$speckit-checklist
```

## Relationship To Project Gates

Keep our project-specific rules:

- Research Gate before design.
- Wrapper-first before New API source changes.
- `Approved for implementation: yes` before tracked-file edits.
- Stop for destructive database, production, payment, secret, and core `vendor/new-api` changes.
- Use `ops/production-gate.sh` for operations/billing-sensitive changes.

Before implementation, you may create or update local ignored working notes under `docs/ai-dev/<YYYY-MM-DD>-<topic>/` and run:

```bash
bash ops/ai-dev-check.sh docs/ai-dev/<YYYY-MM-DD>-<topic>
```

The `tasks.md` approval line remains the local guard for those notes. Durable decisions, verification evidence, and residual risk should be copied into the PR, final response, or maintained runbook rather than committed under `docs/ai-dev/`.
